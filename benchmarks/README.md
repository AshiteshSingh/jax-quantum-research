# JAX Quantum Circuits — Benchmark Suite

Performance benchmarking suite comparing `jax_qsim` against other quantum simulation frameworks.

---

## Benchmark Files

| Script | Description | Runtime |
|---|---|---|
| `n10_rigorous_benchmark.py` | **Primary:** N=10 scaling (4–20q) + gradient (15q). CPU baseline. | ~4 min |
| `cuda_vs_cpu_benchmark.py` | GPU vs CPU scaling. Requires CUDA GPU. | ~10 min |
| `benchmark_27q.py` | 27-qubit cross-framework comparison. Requires GPU + PennyLane. | ~20 min |
| `honest_benchmark.py` | jax_qsim vs PennyLane vs Cirq (4–12q). CPU. | ~5 min |

---

## Methodology

### Warmup Protocol
- **N_WARMUP = 2**: Two warmup calls before timing begins (JIT compilation)
- **N_RUNS = 10**: Ten timed calls post-warmup
- Results reported as **mean ± σ** over all 10 runs

### JIT Retracing Note
For `jax.value_and_grad`, the first timed call may include a residual JIT retrace
(even after 2 warmup runs), because gradient tape construction is deferred until the
first differentiated call. We therefore also report the **9-run stable mean** (runs 2–10)
as the physically meaningful post-JIT figure.

### Platform Requirements
- **Scaling/gradient benchmarks:** CPU-only (no GPU required)
- **GPU benchmarks:** NVIDIA GPU with CUDA 12, JAX CUDA build
- **27q comparison:** PennyLane, Cirq optional dependencies
- **Windows:** All scripts use ASCII-only print and `encoding='utf-8'` JSON output

```powershell
# View results
cat benchmarks/results/n10_benchmark_<timestamp>.json
```
